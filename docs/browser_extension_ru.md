# Браузерное расширение

## 1. Назначение

Расширение предназначено для быстрой проверки подозрительных ссылок, телефонов и выделенного текста через backend AI-Analyst Platform.

Поддерживаемая модель:

- Chromium Manifest V3

Подходит для:

- Google Chrome
- Microsoft Edge
- других Chromium-браузеров с поддержкой MV3

## 2. Состав расширения

- `manifest.json`
- `background.js`
- `content.js`
- `popup.html`
- `popup.js`
- `popup.css`

## 3. Основные функции

- ручная проверка значения в popup
- context menu `Проверить в AI-Analyst`
- непрерывный анализ DOM-страницы
- batch-проверка ссылок и телефонов
- визуальная подсветка совпадений на странице
- хранение режима защиты и API URL в `chrome.storage`

## 4. Установка в браузер

1. Откройте `chrome://extensions` или `edge://extensions`
2. Включите `Developer mode`
3. Нажмите `Load unpacked`
4. Выберите папку `browser_extension`

## 5. Настройка

После загрузки расширения:

1. Откройте popup
2. Проверьте адрес backend API
3. Убедитесь, что используется `http://localhost:8010/api/v1` для локального режима
4. Включите `Continuous Protection`, если нужна постоянная проверка страниц

## 6. Как работает поток проверки

### Popup check

1. Пользователь вводит URL, email, телефон или другой текст
2. Popup отправляет запрос в backend
3. Результат показывается в popup

### Context menu

1. Пользователь выделяет текст на странице
2. Вызывает контекстное меню
3. Расширение отправляет выделенное значение в backend

### Continuous protection

1. Content script обходит DOM
2. Собирает ссылки и телефонные значения
3. Отправляет batch-check
4. Подсвечивает найденные совпадения

## 7. Зависимость от backend

Расширение не содержит локальный blacklist и опирается на API:

- `POST /api/v1/fraud/check`
- `POST /api/v1/fraud/check-batch`

Поэтому для корректной работы должны быть доступны:

- backend API
- корректный API URL в настройках popup

## 8. Рекомендации по ручной проверке

- проверить подсветку известного blacklist URL
- проверить проверку номера телефона
- проверить отключение и повторное включение `Continuous Protection`
- проверить работу context menu на произвольном выделенном тексте
- проверить поведение при недоступном backend
